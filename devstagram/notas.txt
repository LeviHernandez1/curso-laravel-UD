MVC (Modelo Vista Controlador) 
    Modelo 
        - Encargado de todas las interacciones en la base de datos, obtener datos, actualizarlos y eliminarlos.
        - El Modelo se encarga de consultar una base de datos, obtiene la informacion per no la muestra, eso es trabajo de la vista.
        - El Modelo tampoco se encarga de de actualizar la información directamente; es el Controlador quien decide cuándo llamarlo.
    Vista
        - Se encanrga de todo lo que se ve en pantalla (HTML)
        - Laravel tiene un Template Engine LLamado Blade para mostrar los datos.
        - Si utlizamos React, Vue, Angular, Svelte, estos serían tu vista.
        - El modelo consulta la base de datos, pero es por medio del Controlador que se decide que Vista hay que llamar y que datos presentar.
    Controlador
        - Es el que comunica modelo y vista; antes de que el Modelo consulte la base de datos el Controlador es el encargado de llamar un Modelo en especifico.
        - Una vez consultado el Modelo, el controlador recibe esa informacion, manda llamar a la vista y le pasa la informacion.
        - El controlador es el que manda llamar la vista y modelos, que se requieren en cada parte de tu aplicacion.
    Router en MVC
        - Es el encargado de registrar todas las URL's o Endpoints que va a soportar nuestras aplicaciones.

Tipos de Request en HHTTP

En Http existen diferentes tipos de Request o otios de peticiones:
GET, POST, PUT, PATCH, y DELETE.

GET
    Es el mas simple; cuando se visita un sitio web por default es un GET
    Solo se utiliza para recuperar datos pero nunca debe enviar datos.

POST 
    Se utiliza cuando mandas datos a un servidor;esto incluye informacion que llenas en formularios o buscadores.

PUT
    Es utilizado para actualizar un elemento; pero si no existe crea uno nuevo; PUT es un remplazo total de un registro
    Pero debemos enviar una actualizacion de todos los registros.

PATCH 
    Es utilizado para actualizar parcialmente un elemento o recurso, solo un campo de todo un registro.

DELETE
    Se utiliza para eliminar un recurso o elemento.

Controlers en Larave 
    Los controlers van a ayudar a tener un código mejor orgabizado,
    además de una separación mayor en la funcionalidad de las aplicaciones y sitios web.
    Laravel tiene una convencion a la hora de nombrar los metodos de mis controllers conocida como Resourse Controllers.
    Esta convencion ayuda bastante para tener todo mejor organizado.

    Acciones de controladores de Resourse 

    Verbo HTTP      |    URI                    |   Acción  | Ruta
    ----------------------------------------------------------------------
        GET         | /clientes                 |   index   |clientes.index
        GET         | /clientes/create          |   create  |clientes.create
        POST        | /clientes                 |   store   |clientes.store
        GET         | /clientes/{cliente}       |   show    |clientes.show
        GET         | /clientes/{cliente}/edit  |   edit    |clientes.edit
        PUT/PATCH   | /clientes/{cliente}       |   update  |clientes.update
        DELETE      | /clientes/{cliente}       |   destroy |clientes.destroy

Rutas nombradas

    Nos ayudan a poder nombrar rutas para poder tener una mejor forma de tener rutas en codigo al renombrar.
    
    Antes:
    WEB.php     = Route::get('/crear-cuenta', [RegisterController::class, 'index']);
    BLADE.php   = <a href="/crear-cuenta" ></a>

    Despues:
    BEB.php     = Route::get('/crear-cuenta', [RegisterController::class, 'index'])->name('register');
    BLADE.php   = <a action="{{ route('register')}}" ></a>

Validaciones

Sintaxis de las validaciones podemos escribirlas en dos formatos diferentes en el controller.

    1. Con separadores de "|" podemos hacerlo uno sobre el otro en una misma linea:
        $data = $request->validate([
            'name' => 'required|min:5|max:7',
        ]);
    2. Con forma de arreglo entre []:
        $data = $request->validate([
            'name' => ['required','min:5','max:7'],
        ]);

Para desavilitar la validacion de HTML5 por la de el distribuidor se agregan a las etiquetas como form "novalidate"
    Ejemplo: 
        <form href="{{route('register')}}" method="POST" novalidate>

BASE DE DATOS

Migraciones
    Las migraciones se les conce como el control de versiones de tu base de datos; de esta forma se puede crear la base de 
    datos y compartir el diseño con el equipo de trabajo.

    Si deseas agregar nuevas tablas o columnas a una tabla existente, puedes hacerlo con una nueva migracion, si el resultado 
    no fue el deseado, puedes revertir esta migracion.

    Comandos utilices 
        Correr migraciones 
            sail php artisan migrate (Tambien se puede con "sail artisan migrate" )
        Regresar al estado anterior de la migracion realizada
            sail artisan migrate:rollback
        Regresar al estado anterior a ejecutar n cantidad de migraciones
            sail artisan migrate:rollback --step=5
        Solo borra y deshace todas las tablas mediante los métodos down(), dejando la base de datos vacia
            sail php artisan migrate:reset
        Hace exactamente lo mismo pero además vuelve a ejecutar todas las migraciones con migrate de inmediato.
            sail php artisan migrate:refresh
        Para crear una migracion
            sail php artisan make:migration Nombre_migracion (Tambien se puede con "sail artisan make:migration Nombre_migracion")

    Dentro de las migraciones.
        Se crearon dos metodos al crear una nueva migracion, dentro de uno se escribe lo que se crea y en el otro lo mismo pero para eliminarlo
            Metodo Up :
                Se ejecuta cuando corremos el comando " sail artisan make:migration "
            Metodo down:
                Se ejecuta cuando corremos un rollback

Entrar a las bases de datos por terminal

Desde laravel prodemos tener la confirmacion del pasword por medio de el name de ambos inputs deben llamarse asi .

        id="password"
        name="password"
En el blade para el segundo password donde se debe confirmar:
    <input

En el Controller tenemos que indicar la regla de confirmacion.
    $data = $request->validate([
            'password' => 'required|confirmed',
        ]);

Eloquent (El ORM de Laravel)
    ORM (Object Relacional Mapper)
    Laravel incluye su propio ORM que hace muy sencillo interactuar con la base de datos.
    Ya biene instalado en laravel.
    En Eloquent cada tabla, tiene su propio modelo; ese modelo interactua unicamente con esa 
    tabla y tiene las fuinciones necesarioas para crear registros, obtenerlos, actualizarlos y eliminarlos.

Modelos
    Los modelos en Eloquen para las bases de datos se crean con el comando:
    " sail php artisan make:model Nombre_del_modelo " el nombre en singular normalmente.
    Convenciones
        Cuando crear el Modelo CLiente, Eloquent asume que la tabla se va a llamar clientes
        Si el Modelo se llama Producto; Eloquent espera una tabla llamada productos.
        Puede ser un problema llamar a un modelo con ejemplo Proveedor, por que Eloquent espera la tabla rpovedors,
        pero se puede reescribir en el modelo.
    Los modelos se importan en los controladores.
    Es importante que los modelos tengan definido en fillable ya que deben tener los campos
    espcificos en los que se escribira en la base de datos los registros, esto por medidas de seguridad del propio 
    framework
    Seguridad con los password
        Es importante mencionar que en los modelos la seguridad con los pasword se definen aqui.
        Laravel 10 
            Podemos hashear el pasword desde el controller
                use Illuminate\Support\Facades\Hash; // funciones de laravel 
                User::create([
                    // Arreglos asociativos para obtener informacion del objeto es con request
                    'name' => $request->name,
                    //Para poder obtener el Username en minusculas se puede hacer de la siguiente manera 
                    //'username' =>Str::lower($request->username),
                    'username' => $request->username,
                    'email' => $request->email,
                    'password' => Hash::make($request->password), // Aqui se integra la seguridad
                ]);
        Laravel 11
            Podemos hashear desde el Modelo.
                El siguinete cast ayuda a agregar caracteristicas a los datos 
                por ejemplo password nos aporta la proteccion a las constraseñas con hashed
                
                protected function casts(): array
                {
                    return [
                        'email_verified_at' => 'datetime',
                        'password' => 'hashed',
                    ];
                }

Autenticaciones de usuarios
    Laravel ya nos proporciona la autenticacion de usuarios
    Ademas nos proporciona helpers que nos apoyan en la comprobacion de un usuario autenticado.

    Los datos del usuario los guarda en user
    //Autenticar un usuario
        //Helper auth() y funcion para autenticar 
        auth()->attempt([
            'email'=> $request->email,
            'password'=> $request->password,
        ]);
    // Otra forma de autenticar al usuario
        auth()->attempt($request->only('email', 'password'));

Proteccion de rutas con middleware
El middleware auth funciona exactamente como un "guardia de seguridad" en la puerta de la ruta
    Antes se usaba esto en el controlador:
        //Constructor instanciado de PostCOntroller
            public function __construct()
            {   
                // Esto protege que un usuario debe estar autenticado para poder ver el contenido
                $this->middleware('auth');
            }
    Ahora se impelmeta desde la ruta en el archivo web.php
