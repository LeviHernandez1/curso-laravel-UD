MVC (Modelo Vista Controlador) 
    Modelo 
        - Encargado de todas las interacciones en la base de datos, obtener datos, actualizarlos y eliminarlos.
        - El Modelo se encarga de consultar una base de datos, obtiene la informacion per no la muestra, eso es trabajo de la vista.
        - El Modelo tampoco se encarga de de actualizar la información directamente; es el Controlador quien decide cuándo llamarlo.
    Vista
        - Se encanrga de todo lo que se ve en pantalla (HTML)
        - Laravel tiene un Template Engine LLamado Blade para mostrar los datos.
        - Si utlizamos React, Vue, Angular, Svelte, estos serían tu vista.
        - El modelo consulta la base de datos, pero es por medio del Controlador que se decide que Vista hay que llamar y que datos presentar.
    Controlador
        - Es el que comunica modelo y vista; antes de que el Modelo consulte la base de datos el Controlador es el encargado de llamar un Modelo en especifico.
        - Una vez consultado el Modelo, el controlador recibe esa informacion, manda llamar a la vista y le pasa la informacion.
        - El controlador es el que manda llamar la vista y modelos, que se requieren en cada parte de tu aplicacion.
    Router en MVC
        - Es el encargado de registrar todas las URL's o Endpoints que va a soportar nuestras aplicaciones.

Tipos de Request en HHTTP

En Http existen diferentes tipos de Request o otios de peticiones:
GET, POST, PUT, PATCH, y DELETE.

GET
    Es el mas simple; cuando se visita un sitio web por default es un GET
    Solo se utiliza para recuperar datos pero nunca debe enviar datos.

POST 
    Se utiliza cuando mandas datos a un servidor;esto incluye informacion que llenas en formularios o buscadores.

PUT
    Es utilizado para actualizar un elemento; pero si no existe crea uno nuevo; PUT es un remplazo total de un registro
    Pero debemos enviar una actualizacion de todos los registros.

PATCH 
    Es utilizado para actualizar parcialmente un elemento o recurso, solo un campo de todo un registro.

DELETE
    Se utiliza para eliminar un recurso o elemento.

Controlers en Larave 
    Los controlers van a ayudar a tener un código mejor orgabizado,
    además de una separación mayor en la funcionalidad de las aplicaciones y sitios web.
    Laravel tiene una convencion a la hora de nombrar los metodos de mis controllers conocida como Resourse Controllers.
    Esta convencion ayuda bastante para tener todo mejor organizado.

    Acciones de controladores de Resourse 

    Verbo HTTP      |    URI                    |   Acción  | Ruta
    ----------------------------------------------------------------------
        GET         | /clientes                 |   index   |clientes.index
        GET         | /clientes/create          |   create  |clientes.create
        POST        | /clientes                 |   store   |clientes.store
        GET         | /clientes/{cliente}       |   show    |clientes.show
        GET         | /clientes/{cliente}/edit  |   edit    |clientes.edit
        PUT/PATCH   | /clientes/{cliente}       |   update  |clientes.update
        DELETE      | /clientes/{cliente}       |   destroy |clientes.destroy

Rutas nombradas

    Nos ayudan a poder nombrar rutas para poder tener una mejor forma de tener rutas en codigo al renombrar.
    
    Antes:
    WEB.php     = Route::get('/crear-cuenta', [RegisterController::class, 'index']);
    BLADE.php   = <a href="/crear-cuenta" ></a>

    Despues:
    BEB.php     = Route::get('/crear-cuenta', [RegisterController::class, 'index'])->name('register');
    BLADE.php   = <a action="{{ route('register')}}" ></a>
